# 多Agent协作规范

## 1. 角色分工
- **主Agent**: 任务调度、全局决策、结果汇总
- **领域Agent**: 垂直领域专业能力实现
- **工具Agent**: 工具调用、资源操作
- **监控Agent**: 流程监控、异常告警

## 2. 通信协议
- 消息格式: JSON
- 消息ID全局唯一
- 必须包含sender、receiver、type、payload字段
- 异步消息必须有回调地址/响应标识

## 3. 任务流转规则
- 任务分配遵循能力匹配原则
- 子任务完成后必须向主Agent上报结果
- 任务超时自动重试2次，仍失败则告警
- 复杂任务拆分为子任务并行执行

## 4. 冲突处理机制
- 资源抢占采用优先级调度
- 决策冲突由主Agent最终裁决
- 数据冲突采用版本号乐观锁机制
- 异常冲突提交人工审核

## 5. 权限边界
- Agent只能访问所属节点下的资源
- 跨节点访问必须申请临时权限
- 敏感操作必须经过主Agent授权
- 所有操作留痕可审计
