## 根因分析

### Bug 1: SQLite 向量搜索 0 结果
- **现象**: `np.frombuffer()` 读 embedding 报错
- **根因**: `batch_vectorization.py` 用 `json.dumps()` 存 JSON 文本数组，代码假设 binary blob
- **修复**: 改用 `json.loads()` 解析文本，兼容两种格式
- **与 TurboQuant 关系**: ❌ 无关。TQ 是独立压缩层，有自己的 `.tq` 文件

### Bug 2: BM25 中文匹配 0 结果
- **现象**: 查询 "统一搜索引擎" 返回 0 结果
- **根因**: 索引用 jieba 分词（词级，avg 2.43字），查询用 regex 把整串当一 token
- **修复**: 中文 token 展开为 2-gram
- **与 TurboQuant 关系**: ❌ 无关。纯分词策略不一致

### Bug 3: TurboQuant .tq 空文件
- **现象**: `openclaw_memory_8bit.tq` = 0 bytes
- **根因**: 待排查
- **状态**: P1-5 进行中
